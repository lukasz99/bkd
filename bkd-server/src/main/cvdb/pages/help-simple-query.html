The text search supports the following operators:
  • + signifies AND operation
  • | signifies OR operation
  • - negates a single token
  • " wraps a number of tokens to signify a phrase for searching
  • * at the end of a term signifies a prefix query
  • ( and ) signify precedence
To use one of these characters literally, escape it with a preceding backslash (\).

Note: Searches are NOT case-sensitive and check for words present within all fields of the protein/gene records.

Sample queries:

  • SCN1B - Find record annotated with SCN1B word. Note: Requires exact word match.
  • SCN1* - Find record annoted with words starting with SCN1. SCN1A, SCN1B, SCN11 will match. mSCN1B will not.
  • calcium potassium - Find record annoted with both calcium and potassium. Same as calcium + potassium .
  • calcium | potassium - Find record annoted with calcium or potassium.
  • calcium - potassium - Find record annoted with calcium but not potassium.
